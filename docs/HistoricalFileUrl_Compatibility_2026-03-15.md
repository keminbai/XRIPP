# 历史文件 URL 兼容收口（2026-03-15）

## 1. 背景

上传路径稳定化修复完成后，新上传文件已经可以正常：

- 上传成功
- 保存成功
- 再次打开时正常显示

但历史记录仍存在零散异常，表现为：

- 编辑已有记录时，`el-upload` 缩略图不显示
- 旧附件点击预览/下载命中前端本地路径，出现 404
- 个别页面列表图、详情图、文件链接口径不一致

本阶段针对的是“历史数据兼容层”，不是重新修上传底座。

## 2. 根因

根因不是后端写盘失败，而是前端对历史文件 URL 的处理口径不统一：

1. 数据库存的是原始路径，如 `/uploads/...`
2. 新上传流程里，页面通常能直接拿到当前返回结果，因此更容易正常显示
3. 历史记录回显时，有些页面直接把原始 `fileUrl` 填回：
   - 模板 `href`
   - `el-upload` 的 `fileList.url`
   - 列表图 / 详情图 / 下载按钮
4. 在 Nuxt 开发代理环境下，原始 `/uploads/...` 需要转换为 `/api/uploads/...` 才能正确访问后端静态资源

## 3. 本次修复策略

核心原则：

- 保留原始 URL 作为业务/持久化字段
- 额外生成“预览 URL”作为前端显示字段
- 禁止直接把原始 `fileUrl` 覆写成代理路径，避免编辑保存时把存储口径带偏

统一做法：

1. `normalizeUploadMeta(...)`
   - 保留原始 `fileUrl` / `url`
   - 新增 `previewUrl = resolveFileUrl(rawUrl)`
2. `toUploadUserFile(...)`
   - `el-upload` 统一使用 `previewUrl`
3. 模板中的预览、图片展示、下载链接
   - 统一优先使用 `previewUrl`
   - 必要时再通过 `resolveFileUrl(rawUrl)` 兜底

## 4. 本次落地页面

### 4.1 `member/supplier-apply.vue`

修复点：

- 历史附件回显时补充 `previewUrl`
- 步骤页中“已上传”附件链接统一走解析后地址
- `el-upload` 文件列表缩略图回显统一使用解析地址

效果：

- 历史营业执照 / 开户许可证 / 宣传封面 / 公司 PDF / 宣传图可正常打开
- 不影响提交审核时传给后端的原始 `file_url`

### 4.2 `admin/overseas/services.vue`

修复点：

- 宣传图、服务图片、介绍文件统一保留原始 `fileUrl`，新增 `previewUrl`
- 列表缩略图使用解析后地址
- 详情弹窗图片预览和文件下载统一使用解析后地址
- 编辑回显时 `el-upload` 文件列表正常显示历史文件

效果：

- 历史出海服务记录可正常查看图片、下载介绍文件
- 不污染编辑保存的原始文件路径

### 4.3 `admin/overseas/reports.vue`

修复点：

- 封面图、报告文件回显统一生成 `previewUrl`
- `el-upload` 历史文件回显恢复
- 下载动作统一先做路径解析再打开

效果：

- 历史报告封面与报告文件可正常回显/下载
- 报告编辑保存仍沿用原始存储路径

## 5. 风险控制

本次故意没有做“把所有 `fileUrl` 直接改写为 `/api/uploads/...`”，因为那会带来两个风险：

1. 编辑旧记录后再次保存，可能把代理路径写回数据库
2. 前端显示口径和后端存储口径混在一起，后续更难维护

因此本阶段采用“双轨字段”：

- 原始字段：用于持久化和提交
- 预览字段：用于界面显示和打开文件

## 6. 建议复测

优先复测历史记录，而不是新上传记录：

1. 会员侧服务商申请
   - 打开一条旧申请
   - 查看五类附件链接是否正常
   - 检查编辑页 `el-upload` 是否能显示旧文件
2. 后台出海服务
   - 打开旧服务详情
   - 查看宣传图、服务图片、介绍文件下载
   - 进入编辑确认旧文件是否回显
3. 后台行业报告
   - 编辑旧报告
   - 确认封面与报告文件都能回显
   - 点击下载确认能打开文件

## 7. 结论

本阶段解决的是“新上传已稳，但历史记录仍偶发异常”的兼容层问题。  
它不改变上传底座，也不改变数据库结构，而是把历史文件回显、预览、下载三条路径统一到同一套前端解析规则上。
